# SDP Pipeline Standards -- User Instructions (Template)

> **This is a template.** To create a ready-to-use version for your workspace:
> 1. Choose your approach: **direct skills** or **MCP**
> 2. Copy the matching file from `local_deployment/instructions_to_use/`
>    - `user_instructions_skills.md` -- if you uploaded skills to your workspace
>    - `user_instructions_mcp.md` -- if you set up a GitHub MCP connection
> 3. Fill in any remaining placeholders (org, repo, pipeline names)
> 4. Save as `/Users/{username}/.assistant_instructions.md` in your workspace

**Always build SDP pipelines using SQL, not Python.** All tables must be defined as SQL `CREATE OR REFRESH MATERIALIZED VIEW` or `CREATE OR REFRESH STREAMING TABLE` statements in `.sql` files.

When creating or modifying pipeline tables, use the standards below.

## Choose ONE of the following approaches:

---

### Approach A: Direct Skills Reference (no MCP)

> Use this if you have uploaded skills directly to your Databricks workspace
> at `Workspace/.assistant/skills/` or `/Users/{username}/.assistant/skills/`.

When creating or modifying SDP pipeline tables, always use the following skills:

1. **@table-governance** -- Apply table and column documentation standards (COMMENT, TBLPROPERTIES, column descriptions, UC tags). Apply this first for every table.
2. **@sdp-basics** -- Apply SDP naming conventions, audit columns, data quality constraints, and SQL formatting
3. **@pii-management** -- Identify, label, and annotate PII columns in tables containing personal data
4. **@sentiment-analysis** -- Apply when building tables that use AI functions (`ai_analyze_sentiment`, `ai_classify`, `ai_extract`, `ai_summarize`) on free-form text such as customer reviews, support tickets, or transcripts

For every table you create:
- Always apply the table-governance and sdp-basics skills
- If the table contains customer or personal data, also apply the pii-management skill
- If the table derives sentiment, topics, or entities from text via AI functions, also apply the sentiment-analysis skill

## Quick Reminder

- Audit columns: `audit_timestamp`, `source_system` (as LAST columns)
- All tables need: layer prefix, COMMENT, TBLPROPERTIES with `quality` + `owner`
- Add column descriptions for PKs, derived columns, and PII columns
- Bronze: WHERE clause filtering, CLUSTER BY AUTO
- Silver: CONSTRAINT clauses, data_quality_flag
- Gold: ROUND() for money, LIVE.table_name for joins
- PII: Use income_tier/credit_tier derivation, mask emails/phones
