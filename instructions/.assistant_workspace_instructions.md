> **This is a template.** To create a ready-to-use version for your workspace:
> 1. Choose your approach: **direct skills** or **MCP**
> 2. Copy the matching file from `local_deployment/instructions_to_use/`
>    - `workspace_instructions_skills.md` -- if you uploaded skills to your workspace

> 3. Fill in any remaining placeholders (org, repo)
> 4. Save as `Workspace/.assistant_workspace_instructions.md` in your workspace

These workspace-wide instructions enforce data engineering governance across all users. Genie Code applies these rules to every response.

---

## Table Documentation Policy

- Every table MUST have a `COMMENT` clause describing its purpose.
- Every table MUST have `TBLPROPERTIES` with at least `"quality"`, `"owner"`, and `"domain"`.
- Every table containing personal data MUST include `"contains_pii" = "true"` and `"pii_columns" = "<column_list>"` in TBLPROPERTIES.
- PII columns MUST be annotated with inline comments indicating risk level (e.g., `-- [PII: EMAIL - HIGH]`).
- Primary keys, derived columns, and PII columns MUST have column descriptions via `ALTER TABLE ... ALTER COLUMN ... COMMENT`.
- Unity Catalog tags (`quality`, `domain`, `pii`, `data_classification`) MUST be applied to all tables.

## Naming Conventions

- All table names MUST use `lowercase_snake_case` with layer prefixes: `bronze_`, `silver_`, `gold_`.
- No PascalCase, UPPERCASE, kebab-case, or camelCase.

## Audit Columns

- Every table MUST include `audit_timestamp` and `source_system` as the LAST columns in the SELECT.

## Pipeline Language

- All SDP pipelines MUST be written in SQL, not Python.

## Choose ONE of the following approaches:

---

### Approach A: Direct Skills Reference (no MCP)

> Use this if skills have been uploaded to the workspace at
> `Workspace/.assistant/skills/`.

All pipeline authors must use the following skills when creating or modifying tables:

1. **@table-governance** -- Table and column documentation, TBLPROPERTIES, UC tags, PII labeling (apply first)
2. **@sdp-basics** -- Naming, audit columns, data quality constraints, SQL formatting
3. **@pii-management** -- PII detection, labeling, masking, and GDPR patterns
4. **@sentiment-analysis** -- AI-function patterns (sentiment, topic, extraction) for tables built from free-form text
